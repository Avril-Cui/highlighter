# Web Highlighter — Chrome Extension

Highlight text on any webpage. Highlights persist across sessions, synced via Firebase.

## Features

- **Persistent highlights** — survive page reloads and browser restarts
- **Custom confirm key** — Space, Enter, H, or any key you choose
- **Custom colors** — pick any highlight background and text color
- **Hover to remove** — hover over a highlight to see a remove button
- **Reading stats** — pages read and time spent reading today
- **Auth** — email/password or Google sign-in

## Setup (one-time)

### Step 1 — Create a Firebase project

1. Go to [console.firebase.google.com](https://console.firebase.google.com) and click **Add project**
2. Give it a name (e.g. `web-highlighter`) and finish the wizard
3. In the left sidebar, go to **Build → Authentication → Get started**
4. Enable **Email/Password** and **Google** sign-in providers
5. In the left sidebar, go to **Build → Firestore Database → Create database**
   - Choose **Start in test mode** (you can add security rules later)
   - Pick any region

### Step 2 — Get your Firebase config

1. In the Firebase console, click the ⚙️ gear → **Project settings**
2. Scroll down to **Your apps** → click **Add app** → choose the **Web** icon (`</>`)
3. Register the app (nickname doesn't matter), copy the `firebaseConfig` object

Open `firebase-config.js` and fill in your values:

```js
const FIREBASE_CONFIG = {
  apiKey:    "AIzaSy...",          // from firebaseConfig
  projectId: "your-project-id",   // from firebaseConfig
  oauthClientId: "..."            // filled in Step 3 below
};
```

### Step 3 — Set up Google OAuth for the Chrome Extension

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Make sure your Firebase project is selected in the top dropdown
3. Go to **APIs & Services → Credentials → Create Credentials → OAuth 2.0 Client ID**
4. Application type: **Chrome Extension**
5. For **Item ID**: you need your extension's ID (see Step 4 for how to get it, then come back)
6. Copy the **Client ID** (looks like `123456789-abc.apps.googleusercontent.com`)
7. Paste it into `firebase-config.js` as `oauthClientId`
8. Also paste it into `manifest.json` under `"oauth2" → "client_id"`
9. In Firebase console → **Authentication → Sign-in method → Google → Edit** →
   add this Client ID as an **Authorized Client ID**

### Step 4 — Generate icon files

1. Open `icons/generate-icons.html` in Chrome
2. Click each **Download** button and save the files into the `icons/` folder:
   - `icon16.png`
   - `icon48.png`
   - `icon128.png`

### Step 5 — Load the extension in Chrome

1. Go to `chrome://extensions`
2. Toggle **Developer mode** ON (top-right)
3. Click **Load unpacked** and select this folder
4. The extension ID now appears under the extension card — copy it
5. Go back to Google Cloud Console (Step 3) and add this ID to your OAuth client

### Step 6 — Set Firestore security rules (recommended)

In the Firebase console → Firestore → **Rules**, replace the default with:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## Usage

1. Click the extension icon in Chrome toolbar → **Sign In** or **Create Account**
2. In **Settings**, choose your highlight color, text color, and confirm key
3. On any webpage, **select some text**, then press your confirm key (default: Space)
4. The highlight appears instantly and is saved to your account
5. **Hover** over a highlight to see the red **✕ Remove** button
6. View your reading stats in the **Stats** tab
